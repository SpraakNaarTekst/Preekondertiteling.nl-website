Inloggen op de Server PC via VNC

VNC opstarten

VNC staat geïnstalleerd op de beheerderstablet

Als er tijdens de dienst bijvoorbeeld geen tekst doorkomt, of met een lange vertraging, kan het soms zijn dat er op de server PC iets aan de hand is wat de preekondertiteling verstoort. Door op de tablet via VNC in te loggen op de server PC kan de fout verholpen worden zonder tijdens de dienst naar de PC te hoeven lopen.

Door te klikken op het VNC icoontje zoals dat rechts staat, wordt de VNC viewer op de tablet geopend.

Andere tablets zien dit niet, en hebben hier ook geen last van.

Let op: om in te loggen op de server PC is het nodig om binnen hetzelfde (draadloze) netwerk te zitten als de server PC. Vanaf thuis werkt het inloggen op de server via VNC dus niet. Vanaf thuis kan teamviewer gebruikt worden.

Problemen oplossen

Vaak is het probleem een popup-scherm

Dit kan bijvoorbeeld het 'Dragon spellingsvenster' zijn. Na het sluiten van de pop-up wordt de tekst weer op de juiste manier verwerkt en zal de tekst weer op de tablets verschijnen zoals dat gebruikelijk is.

Let op: soms staat het popup-scherm áchter het Preekondertitelings-scherm. De popup kan dan worden gesloten door te klikken op het icoontje onderin de taakbalk en middels rechtermuisknop > afsluiten de popup te sluiten.

Mocht er geen popup te bekennen zijn, maar het probleem aanhouden, dan kan het beste de server PC herstart worden.

Wilt u in alle gevallen het probleem en de eventuele oplossing melden bij Spraak naar Tekst? Dan kan het probleem in de toekomst wellicht verholpen worden en/of de documentatie op dit punt uitgebreid worden.

VNC afsluiten

Let op: sluit alleen VNC af, niet de server PC!

Na het afsluiten van VNC wordt weer het normale tablet-scherm zichtbaar.

De server PC van Preekondertiteling blijft gewoon aanstaan. Andere tabletgebruikers merken alleen dat het probleem is opgelost.
